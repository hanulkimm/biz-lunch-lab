## 한 줄 요청
첨부한 `design-mockup.html`은 우리 사내 웹앱의 **현재 화면**입니다.
지금은 **"AI가 만든 개발 프로젝트" 같은 밋밋하고 템플릿 같은 느낌**이라,
이걸 **실제 상용 서비스 수준의 완성도**로 리디자인해주세요.

---

## 서비스 개요
- **이름:** Biz Lunch Lab (🍱)
- **무엇:** 회사 본부 직원들이 광화문 권역 점심·회식 맛집을 공유하고, AI로 추천받고, 랜덤 런치 모임을 매칭하는 사내 웹앱
- **사용자:** 기업사업본부 직원 169명 (20~40대 직장인)
- **플랫폼:** PC 웹 (데스크톱 우선)
- **언어:** 한국어

## 지금 문제점 (이런 느낌을 벗어나고 싶음)
- 기본 시스템 폰트 + 어디서나 보는 파란색(#4f7cff) → 개성·브랜드 없음
- 컴포넌트가 부트스트랩 튜토리얼처럼 밋밋함
- 위계·여백·디테일이 단조로워 "실서비스"가 아니라 "과제물" 느낌
- 이모지를 아이콘처럼 씀 (🗺️🤖🎲)

## 원하는 방향
"실제 상용 웹사이트처럼." 구체적으로:
- **명확한 비주얼 아이덴티티** — 이 서비스만의 컬러 팔레트(브랜드 컬러 1~2개 + 중성 톤), 로고/워드마크 처리
- **타이포그래피 위계** — 한글 웹폰트(예: Pretendard) 기반, 크기·굵기·자간이 의도적으로 설계됨
- **깊이와 위계** — 적절한 그림자/보더/여백 리듬으로 정보 우선순위가 보임
- **제대로 된 아이콘** — 이모지 대신 일관된 아이콘 세트(라인 아이콘 등)
- **마이크로 디테일** — hover/active 상태, 부드러운 전환, 빈 상태/로딩 상태 디자인
- **카드·리스트·폼의 완성도** — 실제 서비스의 식당 카드/리뷰 카드처럼
- 분위기 톤은 자유롭게 제안해주세요 (예: 신뢰감 있는 모던 / 따뜻한 푸드 / 깔끔한 미니멀 등). 한 방향으로 밀어붙인 강한 시안을 보고 싶습니다.

## 화면 목록 (mockup의 9개)
1. 로그인 / 2. 회원가입 — 담당·팀 드롭다운 + 이름 + 4자리 PIN
3. 메인 지도 — 헤더 + 지도(마커) + 식당 정보 패널(별점/태그/리뷰)
4. AI 챗봇 패널 — 대화형 맛집 추천
5. 리뷰 작성 — 식당 검색 → 별점 + 태그 멀티선택 + 코멘트
6. 마이페이지 — 내 리뷰 목록 (수정/삭제)
7. 메뉴 룰렛 — 8카테고리 스피너 + 추천 결과
8. 랜덤 런치 — 취향 신청 + 그룹 매칭 결과
9. 관리자 — 회차 관리 + 매칭 실행 + PIN 리셋

## 공통 컴포넌트 (디자인 시스템으로 잡아주세요)
- 버튼 (primary / secondary / danger)
- 입력 (text / select / textarea), 4자리 PIN 입력
- 카드 (식당 카드, 리뷰 카드)
- 태그 칩 (선택 가능), 별점, 관리자 뱃지
- 슬라이드 패널 (지도 위 오버레이)
- 하단 탭바 (5개: 지도/AI/룰렛/런치/마이페이지)
- 채팅 말풍선

## 산출물로 받고 싶은 것
1. **디자인 시스템** — 컬러 토큰, 타이포 스케일, 핵심 컴포넌트 스타일 (CSS 변수 형태면 가장 좋음)
2. **리디자인된 화면들** — 최소 로그인·지도·리뷰작성·챗봇 4개는 렌더 가능한 아티팩트로
3. 가능하면 위 mockup과 같은 **단일 HTML/CSS**로 (그래야 실제 코드로 옮기기 쉬움)

## 기술 메모 (claude.ai가 알아두면 좋은 것)
- 실제 앱은 **React + Vite**. 이 디자인을 받아서 개발자(나)가 React 컴포넌트에 이식할 예정.
- 그러니 **순수 CSS / CSS 변수 / 시맨틱 마크업** 위주로, 특정 프레임워크에 종속되지 않게 부탁.
- 외부 폰트/아이콘은 CDN으로 (Pretendard, 라인 아이콘 세트 등).
- 지도 자체는 카카오맵(실서비스에서 렌더)이라, 목업에선 영역만 잡으면 됨.

## (참고) 컬러·콘텐츠 사실
